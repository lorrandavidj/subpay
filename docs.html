PayZap API

A API REST da PayZap permite que você integre pagamentos Pix, gerencie cobranças e automatize sua operação financeira de ponta a ponta.

Base URL
https://api.payzap.com/v1
Formato

Todos os requests e responses são em JSON. O header Content-Type: application/json é obrigatório para requests com body.

Rate Limiting

Os limites por plano:

PlanoRequests/minRequests/dia
Básico6010.000
Pro300100.000
Enterprise1.000Ilimitado
Idempotência

Para requests POST, envie o header Idempotency-Key: <uuid> para garantir que requests duplicados não gerem cobranças duplicadas.

Autenticação

A PayZap usa Bearer tokens para autenticar requests. Inclua sua chave de API em todas as chamadas.

Header de autenticação
Authorization: Bearer sk_live_xxxxxxxxxxxxxxxxxxx
Tipos de chave
PrefixoAmbienteDescrição
sk_live_ProduçãoProcessa pagamentos reais. Guarde com segurança.
sk_test_SandboxPara desenvolvimento e testes. Não gera cobranças reais.

⚠️ Nunca exponha suas chaves de API no frontend ou em repositórios públicos. Use variáveis de ambiente.

Ambientes

A PayZap oferece dois ambientes isolados: produção e sandbox.

AmbienteBase URLDescrição
Produçãohttps://api.payzap.com/v1Pagamentos reais, chave sk_live_
Sandboxhttps://sandbox.api.payzap.com/v1Testes, chave sk_test_
Comportamento no Sandbox

No sandbox, use valores especiais para simular respostas:
• R$ 10,00 → Aprovação imediata
• R$ 11,00 → Expiração após 30s
• R$ 12,00 → Cancelamento

Criar Cobrança
POST /charges

Cria uma nova cobrança Pix e retorna o QR Code e o código copia-e-cola para o pagador.

Parâmetros
CampoTipoObrigatórioDescrição
amountintegerSimValor em centavos (ex: 9900 = R$ 99,00)
descriptionstringSimDescrição da cobrança (max 150 chars)
customer.namestringNãoNome do pagador
customer.emailstringNãoE-mail do pagador
customer.tax_idstringNãoCPF/CNPJ (apenas dígitos)
expires_inintegerNãoExpiração em segundos (padrão: 3600)
metadataobjectNãoDados customizados (max 10 chaves)
Consultar Cobrança
GET /charges/:id

Retorna os dados de uma cobrança específica pelo ID.

Parâmetros de path
CampoTipoObrigatórioDescrição
idstringSimID da cobrança (formato: chg_xxxxxxxxxx)
Listar Cobranças
GET /charges

Retorna uma lista paginada de cobranças. Suporta filtros por status, data e valor.

Query parameters
CampoTipoObrigatórioDescrição
statusstringNãopaid | expired | cancelled | pending
fromstringNãoData inicial ISO 8601 (ex: 2025-01-01)
tostringNãoData final ISO 8601
limitintegerNãoItens por página (1–100, padrão: 20)
pageintegerNãoNúmero da página (padrão: 1)
Reembolsar Cobrança
POST /charges/:id/refund

Inicia o reembolso de uma cobrança paga. O valor é devolvido via Pix em até 7 dias úteis.

Parâmetros
CampoTipoObrigatórioDescrição
amountintegerNãoValor do reembolso em centavos. Se omitido, reembolso total.
reasonstringNãoMotivo do reembolso (max 200 chars)
Consultar Saldo
GET /balance

Retorna o saldo atual da conta, incluindo disponível para saque, a receber e retido.

Solicitar Saque
POST /withdrawals

Solicita um saque do saldo disponível para uma chave Pix cadastrada.

Parâmetros
CampoTipoObrigatórioDescrição
amountintegerSimValor em centavos (mínimo: 500)
pix_keystringSimChave Pix de destino (CPF, e-mail, telefone ou aleatória)
Listar Saques
GET /withdrawals

Retorna o histórico de saques paginado.

Cadastrar Webhook
POST /webhooks

Registra uma URL para receber notificações de eventos em tempo real.

Parâmetros
CampoTipoObrigatórioDescrição
urlstringSimURL HTTPS de destino (deve retornar 200)
eventsarraySimLista de eventos para assinar (ex: ["charge.paid"])
secretstringNãoSecret para validação de assinatura HMAC
Listar Webhooks
GET /webhooks

Retorna todos os webhooks cadastrados e suas estatísticas de entrega.

Remover Webhook
DELETE /webhooks/:id

Remove permanentemente um webhook cadastrado.

Listar Eventos
GET /events

Retorna o log de eventos da sua conta, com suporte a filtros e paginação.

Guia de Webhooks

Webhooks permitem que sua aplicação seja notificada em tempo real quando eventos importantes ocorrem na PayZap.

Eventos disponíveis
charge.paidCobrança paga com sucesso
charge.expiredCobrança expirou sem pagamento
charge.cancelledCobrança cancelada
charge.refundedReembolso processado
withdrawal.paidSaque processado
withdrawal.failedFalha no saque
Validando a assinatura

Cada request de webhook inclui o header X-PayZap-Signature com um HMAC-SHA256 do body usando seu secret. Valide sempre antes de processar.

Erros

A API usa HTTP status codes para indicar sucesso ou falha. Erros retornam um objeto JSON com error.code e error.message.

HTTPCódigoDescrição
400invalid_requestParâmetros inválidos ou ausentes
401unauthorizedChave de API inválida ou ausente
402insufficient_fundsSaldo insuficiente para saque
403forbiddenOperação não permitida para este plano
404not_foundRecurso não encontrado
409duplicate_requestRequest duplicado (Idempotency-Key já usada)
422charge_expiredCobrança já expirada
429rate_limitedLimite de requests excedido
